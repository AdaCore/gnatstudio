GPS Internal Beta

Version 0.7.0

2002/03/19

ACT Europe

Welcome to the third GPS beta

This is the third beta version of GPS.

Here is an overview of the changes since version 0.6.0:

  • GPS is now able to perform periodic saves of files. The timeout is configurable.
  • New menu File->Console->Clear to clear the contents of the console.
  • Selected error message is now highlighted in the console.
  • The default project can now be saved.
  • Better integration of the debugger.
  • The list of main units for a project can now be edited. It is possible to select the main unit to compile directly from the menus.
  • It is now possible to create a directory when selecting the object dir.
  • Better automatic indentation.
  • Regression fixed in handling of "Revert Switches" in the switch editor.
  • Regression fixed in handling of compilation error messages.
  • Fixed handling of recursive directories in the source selection.
  • The naming scheme is now properly recomputed.
  • Scrolling of the source editor when displaying the contextual menu removed.
  • Fixed loading of the desktop.
  • Better handling of focus and cursor in source editor
Here is an overview of the changes since version 0.5.0:
  • When quitting, GPS now asks in a more user friendly fashion whether to save the files
  • Better HTML version of the TODO file
  • The Help widget can now be scrolled/navigated through the keyboard
  • A new menu Build->Custom... to launch any make command
  • A new menu File->Save->Save All is provided
  • Menu Navigate->Goto Line... implemented
  • Search in current source file implemented
  • It is now possible to create a project by filling only the first page of the project wizard, and clicking Forward for the other pages
  • Better handling of incomplete installations (e.g missing files)
  • Goto declaration and Goto body are now separated
  • The same file selector is now used everywhere
  • The file view now gives information about the root directory
  • Command line options --help and --version now recognized.
  • Proper handling of new files (in particular on the command line)
  • Do not produce error message when gnatls can't be found on the path
  • Various fixes in the MDI, in particular when closing windows
  • The console window can no longer be destroyed
  • The file view no longer crashes GPS when quitting too early
  • Busy cursor is properly reset in the call graph
  • The Solaris version is more stable
  • Bugs fixed in the file selector
  • Better handling of xrefs in GNAT run time and non project files
  • Better handling of files with multiple separate routines
  • Better handling of wrong external commands
  • Fixed bug when loading a project with the project wizard
  • The build progress bar is no longer changing the size of the windows
  • The log associated with a check in is now automatically saved before doing a check in
  • Files saved under GPS now always have a terminating newline
  • Fixed bug in the handling of Undo/Redo in the editor, associated with the use of Ctrl-K
  • Line and column numbers are now properly computed, and the box displaying this information has a fixed size
  • Fixed handling of default vs file specific switches

Here are the functionalities that are implemented at this stage:

  • File Open/Save/Close
  • Basic source editing
  • Search
  • Source highlighting
  • Automatic indentation
  • Unlimited Undo/Redo
  • Cut/Copy/Paste
  • AUnit macros
  • Basic integration with Emacs and vi
  • Source Navigation, including:
    • Goto Declaration
    • Goto Body
    • Goto Previous/Next Location (Unlimited)
    • List all references to an entity in the project
  • All project related operations, in particular:
    • Default project
    • Project explorer with multiple views:
      • A view Project/Directory/Source/Entities
      • A view File System/Directory/Source/Entities
    • Project Wizard (to create projects easily)
    • Add/Remove project dependencies
    • Add/Remove source directories/files
    • Creation/Use of configuration variables
    • Set/Change Object Directory
    • Switch Editor (per project or per file)
    • Naming Scheme
    • Project Hierarchy Browser
    Full support for Ada is provided, some support for C/C++ as well.
  • Source dependency browser.
  • Call graph (what calls what, what is called by what)
  • Integrated debugging, providing all the functionalities of GVD
  • Basic CVS support, including:
    • Status of files
    • Check in
    • Check out
    • Edit log
    • Add/Remove file
    • View log
    • Visual Diff
  • Compile/Build
  • Check file syntax
  • Jump from error message to source editor
  • Visual Diff between Files
  • Automatic generation of body
  • Source Reformatting
  • Window manipulation
  • Integrated Help for GNAT tools

Many improvements are already on the TODO list, so we strongly suggest that you take a look at this file before submitting suggestions for enhancements.
